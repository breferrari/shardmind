### Changed (v6 layout: HookContext extensions + post-hook re-hash)

Closes [#75](https://github.com/breferrari/shardmind/issues/75). Track 3 of [#70](https://github.com/breferrari/shardmind/issues/70) (v6 contract). Spec: [`docs/SHARD-LAYOUT.md ¬ßHooks, state, and re-hash semantics`](docs/SHARD-LAYOUT.md). **Breaking** for any in-flight shard authoring against the v0.1-pre-#75 `HookContext` shape ‚Äî v0.1 has no published shards, so no migration path is needed. Resolves the forward-reference left in #93's CHANGELOG entry.

- **`HookContext` gains three required fields** (`source/runtime/types.ts`): `valuesAreDefaults: boolean`, `newFiles: string[]`, `removedFiles: string[]`. Required (not optional) so authors get a typed signal that v6 always provides them; the older 4-field ctx still works because the new fields are simply unused by older hook code.
- **`valuesAreDefaults`** ‚Äî pure function `source/core/values-defaults.ts::valuesAreDefaults(values, schema)`. Deep-equal the user values map against the would-be-default map (literal defaults + computed defaults resolved against the literal-default map first). Strict structural equality ‚Äî array order is significant. Computed-default coercion failures are swallowed; result is `false`. Encodes Invariant 2 (hooks that modify managed files must no-op when this is true).
- **`newFiles` / `removedFiles`** ‚Äî fed from `summary.addedFiles` / `summary.deletedFiles` on the update path. New `addedFiles: string[]` field on `UpdateSummary` (`source/core/update-executor.ts`) populated only for `UpdateAction.kind === 'add'`; `overwrite`, `auto_merge`, `restore_missing`, and conflict resolutions excluded (those paths were already in `state.files`). Empty on clean install per spec line 130. Encodes Invariant 3 (post-update hooks are additive-only by default).
- **Post-hook re-hash** ‚Äî new engine function `source/core/state.ts::rehashManagedFiles(vaultRoot, state)` reads every managed file under `mapConcurrent(16)`, recomputes sha256, and returns a new `ShardState`. Per-file ENOENT / EACCES tolerated. Both command machines (`source/commands/hooks/use-{install,update}-machine.ts`) call it after the hook subprocess returns ‚Äî success OR failure ‚Äî and `writeState` the result only when at least one entry changed / went missing / failed. Makes Invariant 2 observable: a hook that legitimately edits a managed file produces zero spurious drift on the next `shardmind` status run.
- **Adversarial cases covered** in `tests/unit/values-defaults.test.ts` (15 cases ‚Äî sentinel defaults, whitespace / case / type-coercion, computed-default chain, computed-default coercion failure, missing-default defensive path, extra-keys ignored, zero-values trivial); `tests/unit/state.test.ts` rehash block (9 cases ‚Äî unchanged, modified-single, mixed-modify-delete-unmanaged-add, hook-deleted via `missing`, ignore-unmanaged, empty-set, FileState-fields preserved, 50-file concurrency, POSIX-only EACCES via `chmod 0o000`); three E2E scenarios in `tests/e2e/cli.test.ts` (full ctx round-trip via `.hook-ctx.json`; `valuesAreDefaults: true` end-to-end with literal-default values; post-hook re-hash via `Home.md` edit + sha256 assertion against `state.files['Home.md'].rendered_hash`); one new integration scenario in `tests/integration/update.test.ts` pinning that `summary.addedFiles` carves out only `UpdateAction.kind === 'add'` paths (overwrite + add scenario asserts `addedFiles` excludes the overwrite).

### Changed (v6 layout: schema defaults + drop `partials`)

Closes [#74](https://github.com/breferrari/shardmind/issues/74). Track 2 of the v6 contract work tracked in [#70](https://github.com/breferrari/shardmind/issues/70). Spec: [`docs/SHARD-LAYOUT.md ¬ßValues, schema, and modules`](docs/SHARD-LAYOUT.md). **Breaking** for any in-flight shard authoring ‚Äî v0.1 has no published shards, so no migration path is needed. Resolves the forward-reference left in the #73 entry below: `mod.partials` is now fully gone (validator + type), and `parseSchema` rejects schemas with values lacking a `default`.

- **`parseSchema` rejects values missing `default`**. The check runs against the raw YAML (post-zod, the `default: z.unknown().optional()` schema strips missing keys, collapsing missing-vs-undefined). Empty/falsey literal defaults (`""`, `false`, `0`, `[]`) are accepted as long as they match the value's `type` ‚Äî the presence rule is satisfied by the key existing, not by any "no default" semantics. `null` is rejected by the type-match rule (next bullet) since it doesn't match any of the six value types. Error code `SCHEMA_VALIDATION_FAILED`; the message lists every offending key (not just the first). Source: `source/core/schema.ts`.
- **`partials` field removed**. `ModuleDefinitionSchema.partials` (`source/core/schema.ts`) and `ModuleDefinition.partials?: string[]` (`source/runtime/types.ts`) deleted. Zod object schemas are non-strict, so v5 shards still shipping `partials: [...]` parse cleanly ‚Äî the field is silently discarded. Pinned by a regression test in `tests/unit/schema.test.ts`.
- **v6-only update path: dead `newRequiredKeys` branch dialed back**. The integration test `prompts only for genuinely missing new required values` is now obsolete under the v6 contract (every value has a default, so `computeSchemaAdditions().newRequiredKeys` always returns `[]`). Rewritten as the v6 negative ‚Äî adding a new value with a default does not contribute to `newRequiredKeys`, pinning the no-prompt branch.

  - `tests/fixtures/schema/invalid-{missing-group,reserved-name,select-no-options,min-gt-max}.yaml`: gain `default` so each fixture continues to fail on its intended check (the new presence check runs last).

### Added (post-install / post-update hook runtime)

Closes [#30](https://github.com/breferrari/shardmind/issues/30). `shard.yaml` `hooks.post-install` / `hooks.post-update` declarations are now actually executed at the end of install / update; previously the engine only *detected* the file and deferred execution (Summary rendered a yellow "not executed" notice). Unblocks Milestone 5 / [#14](https://github.com/breferrari/shardmind/issues/14) ‚Äî obsidian-mind v6 ships a real `post-install.ts` that does first-run `git init` + scaffolding.

- **`source/core/hook.ts` ‚Äî `executeHook`**: subprocess-backed runtime. Spawns `node --import <tsx loader> dist/internal/hook-runner.js <hookPath> <ctxFile>` with `cwd: ctx.vaultRoot` and a JSON-serialized `HookContext` delivered via a 0o600 tempfile (env-var would truncate over the 32 KB Windows cap; stdin would entangle with `source/core/cancellation.ts`). Captures stdout + stderr separately, caps each stream at 256 KB with a `[‚Ä¶ truncated, N bytes discarded]` marker, and forwards chunks live through `onStdout` / `onStderr` callbacks for the streaming UI. Never throws ‚Äî every failure mode (tsx missing, spawn error, timeout, abort, non-zero exit, hook throw) maps to a `HookResult` variant the caller surfaces in the summary.
- **`source/internal/hook-runner.ts`** (new): tiny ESM subprocess entry point. Reads argv, `pathToFileURL`-imports the hook, awaits `mod.default(ctx)`, exits 0/1. Zero Ink / React / Pastel imports ‚Äî cold-start path. Emitted via a fourth tsup entry block at `dist/internal/hook-runner.js` and exposed via a new `./internal/hook-runner` subpath in `package.json`'s `exports` so `require.resolve` finds it regardless of how the package was installed.
- **Per-shard timeout override**: `shard.yaml` `hooks.timeout_ms` (integer, validated 1_000..600_000). Default `DEFAULT_HOOK_TIMEOUT_MS = 30_000` when absent. Enforcement is soft-SIGTERM + 2s grace + SIGKILL. Timeouts surface as `HookResult.failed` with a "timed out after Ns" message and any captured output preserved. `source/core/manifest.ts` + `source/runtime/types.ts`.
- **AbortSignal cancellation**: command machines thread an `AbortController` into `executeHook`. The `useSigintRollback` cleanup fires `.abort()` on every Ctrl+C so the child dies cleanly (the parent's existing SIGINT handler still wins; this is layered on top). Parent SIGINT during the hook phase does NOT trigger rollback ‚Äî state.json is already on disk, per the Helm-style non-fatal contract (ARCHITECTURE ¬ß9.3). Result-decision order inside `executeHook` is cancel > timeout > spawn-error > exit so the user-facing message names the root cause rather than the symptom.
- **Streaming UI ‚Äî new `running-hook` phase** in both `useInstallMachine` and `useUpdateMachine`. Carries a live output buffer capped at 64 KB (tighter than the core cap ‚Äî this buffer lives in React state and re-renders per chunk). Rendered by the new `source/components/HookProgress.tsx` ‚Äî spinner + stage-aware heading + tail-of-last-12-lines. Split on both LF and CRLF so Windows-authored hooks tail cleanly.
- **Summary + UpdateSummary four-branch hook rendering**: `null` ‚Üí nothing, `deferred` (dry run only) ‚Üí dim "skipped" note, `ran + exit 0` ‚Üí green "completed" + labeled stdout + stderr blocks, `ran + exit !=0` (or `failed`-derived) ‚Üí yellow warning with exit code + both captured streams. Warning copy is explicit that install/update succeeded ‚Äî matches the non-rollback contract. `HookSummary` gains an optional `stderr` field; `summarizeHook` folds `failed.message` into stderr prefixed with "hook <reason>" so the two paths render identically.
- **Hook environment**: child inherits parent env plus `SHARDMIND_HOOK=1` and `SHARDMIND_HOOK_PHASE=post-install|post-update` for hook-side branching. Documented in `docs/AUTHORING.md ¬ß6`.
- **Dependency**: `tsx@^4.21.0` added to runtime deps (~6 MB; gated by our existing `node >=22` engine). The loader transpiles TypeScript on the fly in the hook subprocess so shard authors don't need a compile step.
- **Tests: 574 ‚Üí 627 (+53)**. New coverage in `tests/unit/hook.test.ts` (17 subprocess-runtime cases ‚Äî success, throws, process.exit, syntax errors, timeouts, aborts, stream separation + independent caps, env passthrough, ctx round-trip, cleanup, UTF-8 cap-boundary, pre-aborted-signal cleanup, SIGTERM-ignoring hook triggers SIGKILL fallback) plus 6 `tailAtUtf8Boundary` unit cases; `tests/unit/manifest.test.ts` (7 timeout_ms validation cases ‚Äî bounds, non-integers, optional); new `tests/component/HookProgress.test.tsx` (6 cases); new `tests/component/HookSummarySection.test.tsx` (9 four-branch-render cases covering both stages); `tests/integration/install.test.ts` (2 ‚Äî happy + thrown-hook-with-install-intact); `tests/integration/update.test.ts` (1 ‚Äî previousVersion threading); `tests/e2e/cli.test.ts` (2 scenarios ‚Äî real-run with dedicated stub + custom tarball carrying a live hook, plus a `--dry-run` scenario pinning the machine-level `deferred` routing so a future regression back to `null`-in-dry-run surfaces loudly).
- **Docs**: `docs/AUTHORING.md ¬ß6` rewritten to drop the "deferred per #30" hedge and document env tags + timeout override + 256 KB caps + Windows cancellation semantics + stdout/stderr ordering guarantees. `docs/ARCHITECTURE.md ¬ß9.3` ratifies the contract with execution-layer details. `docs/IMPLEMENTATION.md ¬ß4.16` is new (full execution algorithm + result-decision order + ctx-transport rationale + dependencies); `¬ß4.3` gains a `timeout_ms` schema entry; update-flow ¬ß4.12 step 6 describes the live runtime instead of the deferred placeholder. `CLAUDE.md` project tree gains `source/internal/hook-runner.ts` + `source/components/HookProgress.tsx`; tech-stack table gains `tsx`; module-boundary section explains the `source/internal/` rule.

### Fixed (hook runtime ‚Äî post-implementation harden, rounds 1 + 2)

Three-agent audit (adversarial + correctness + docs) across the hook-runtime diff, run to dry on round 3. Every finding fixed in this PR per the "no out of scope" directive:

- **UTF-8 multibyte truncation safety.** The `appendCapped` stream-cap code in `source/core/hook.ts` sliced `Buffer.from(chunk).slice(0, remaining).toString('utf-8')` wherever the 256 KB cap landed ‚Äî which inserted a U+FFFD replacement character whenever the cut fell mid-multibyte (every emoji / CJK output near the cap) and miscounted `dropped` bytes by 1-3. Added `lastCompleteUtf8Boundary(buf, offset)` ‚Äî walks back up to 3 continuation bytes to the nearest complete code-point boundary. Companion `tailAtUtf8Boundary(s, capBytes)` exported for the UI-side streaming buffer in `source/commands/hooks/shared.ts::appendHookOutput`, which previously character-sliced (UTF-16 code-unit width) a byte-declared cap ‚Äî letting emoji output expand the React state 2-4√ó past the intended 64 KB. Both walkers bounded by RFC 3629's 4-byte max.
- **Tempfile write hardened with `flag: 'wx'`.** `source/core/hook.ts` writes the `HookContext` JSON under `os.tmpdir()` with a random 8-byte suffix + 0o600 mode. Adding `O_EXCL` via the `wx` flag closes the TOCTOU window where a hostile actor with write access to a shared tmpdir could race-symlink the resolved name before our write. Belt-and-braces with the random-suffix entropy already in place.
- **E2E + integration test cleanup.** The `post-install hook` describe block in `tests/e2e/cli.test.ts` leaked a `shardmind-e2e-hook-*` staging directory (~200 KB each run); now cleaned up in `afterAll`. Integration-test `afterEach` blocks in `tests/integration/install.test.ts` and `update.test.ts` now use `{ maxRetries: 5, retryDelay: 100 }` on `fsp.rm` ‚Äî matches the pattern unit-hook tests adopted, absorbs the Windows handle-lingering window where a SIGTERM'd subprocess still holds its `cwd` lock for a few ms.
- **Build consistency.** `tsup.config.ts`'s fourth entry (hook-runner) now uses `target: 'node18'` to match the other bundles. Node-engine gate stays `>=22`; the build itself doesn't need to emit node22+ JS.

Reviewed + left intentionally (with rationale in commit messages):

- The `appendHookOutput` generic cast (`prev as unknown as RunningHookPhase`) is load-bearing for cross-machine reuse ‚Äî union-intersection at the call site provides runtime safety; cast is documented in place.
- Pre-existing "swallow" / "best-effort" comments in `executeHook` (tempfile cleanup race, double-kill on already-exited child) describe async-exit races where no user action is possible, not careless error swallowing.
- Hook ANSI escape sanitization: rejected as out-of-scope. Hook authors can opt into colored output; harmful cursor-movement codes are a UX bug, not a security bug. `strip-ansi` would add a dep without clear signal.
- HookContext JSON round-trip lossiness (`undefined` / `Date` / `BigInt`): not a bug ‚Äî shard values load from YAML which doesn't produce those types. Documented as an invariant in `docs/AUTHORING.md ¬ß6` rather than engineered around.

### Fixed (harden pass)

- **Binary assets no longer corrupt on `shardmind update`.** `detectDrift` was reading every tracked file as UTF-8 and hashing the decoded string, while install-executor hashed raw bytes. Copy-origin binaries (images, PDFs, any non-UTF-8 asset) hit an unpaired byte, Node replaced it with U+FFFD on decode, and the sha256 diverged from install-time ‚Äî which classified the file as `modified` on first status check and then ran the three-way merge on corrupted UTF-8 projections, destroying the binary. `source/core/drift.ts` now reads as `Buffer` and hashes bytes; text files are byte-identical across both paths. Regression-guarded in `tests/unit/drift-classification.test.ts`.
- **`shardmind update` no longer silently overwrites untracked user files** at paths the new shard newly introduces. The planner now detects on-disk collisions for `add` actions and routes them through `DiffView` with a `preexisting: true` flag; the executor's `keep_mine` / `skip` resolutions leave the user's file on disk AND untracked, so we never silently adopt content the user didn't opt in to manage. `source/core/update-planner.ts`, `source/core/update-executor.ts`.
- **`hashValues` now terminates on cyclic value graphs** (YAML anchor cycles like `a: &x { self: *x }`). Previously the recursive `stableJson` walk would stack-overflow on hostile input. `source/core/install-planner.ts` adds a `WeakSet` cycle guard; matches the pattern `source/core/renderer.ts :: encodeStringLeaves` already uses for the Nunjucks recovery path.
- **Frontmatter `path_match` glob honors shell semantics.** `brain/*.md` no longer matches `brain/sub/deep/note.md` ‚Äî single `*` stops at `/` as every shell does. Opt in to recursive matching with `**`. `source/runtime/frontmatter.ts`.

### Fixed (Copilot review ‚Äî 3 real regressions caught after rounds 1-7)

Copilot's PR review surfaced three issues the 7-round loop missed. First two are poster children for "self-verification failures are normal" ‚Äî I had a cycle-guard test but no sibling-sharing test, and a file-collision test but no directory-collision test.

### Changed

- `InstallProgress` renamed to `CommandProgress`; both install and update commands render it. Callers should import from `source/components/CommandProgress.js`.
- `mapConcurrent` moved from `source/core/drift.ts` to `source/core/fs-utils.ts` so planner and executor can share the bounded-concurrency primitive.
- `CommandFrame` extracted ‚Äî was `RootFrame` in `install.tsx` and `Frame` in `update.tsx`. Single component owns the dry-run banner + keyboard legend.
- `ShardMindError.code` typed as `ErrorCode` instead of free-form `string` (compile-time check of every call site).
- `detectDrift` now runs per-file reads in parallel via `Promise.all`, and runs the orphan scan in parallel with the classification.
- `source/runtime/errno.ts` centralizes `errnoCode` / `isEnoent`; 8 copies of the `err instanceof Error && 'code' in err ? ...` pattern collapsed.

### Fixed

- `rename` migration no longer silently overwrites an existing value at the target key. If both the old and new keys have values, the migrator warns and keeps both (the user picks which to drop manually). Previously this was a silent data-loss bug found by the adversarial audit.
- `createBackupDir` allocates distinct backup directories even when called twice in the same millisecond. Previous timestamp stripped the fractional seconds, so two near-simultaneous updates could share a backup dir and clobber each other's rollback snapshots.
- `keep_as_user` decisions now untrack the file from `state.files` on apply. The file stays on disk (as the user asked), but the engine no longer considers it managed on future updates ‚Äî matches the UI's "Keep my edits (untrack)" label.
- `DiffView` splits on `/\r?\n/` everywhere to match `differ.ts`'s canonical line splitter. Previously, CRLF content left `\r` in the output strings and Ink's renderer treated them as cursor-moving carriage returns, corrupting the terminal display.
- `conflictFromDirect` now receives the new-shard tempdir and produces a cache-relative `templateKey` instead of an absolute path. The fallback fires only on corrupt / missing cached templates, but when it did fire the state it wrote couldn't be re-used by subsequent updates.
- SIGINT now runs the shard tempdir cleanup regardless of which phase was active. Previously, cancelling with Ctrl-C during the wizard/prompt phases left the extracted tarball in the OS temp directory until the next reboot.
- CRLF on Windows-saved user files no longer produces spurious conflicts against LF-rendered base/ours.
